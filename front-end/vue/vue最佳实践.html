# vue最佳实践

用 vue-cli2 创建的项目把webpack配置文件向外暴露出来，对开发者友好，但是对新人不友好，所以vue-cli3创建的项目把webpack配置文件隐藏起来了

可以使用指令

vue inspect 在命令行查看webpack的所有配置

inspect 后面跟参数可查看对应区域的内容

  • --rules 查看loader 如：vue inspect --rules

    • 把rules改成rule后面在跟指令即可查看该loader的细节配置 如：vue inspect --rule vue

  • --plugins 查看插件配置

    • 把plugins改成plugin后面在跟指令即可查看该plugin的细节配置

虚拟dom是一个js对象，它能和真实的dom进行映射。作为一个中间层

模板编译

模板编译的主要目的就是将模板（template）转换为渲染函数（render）

vuejs项目中的一些最佳实践

  • 项目配置

    vuecli3创建的项目把webpack配置隐藏起来了，可以在根目录下创建一个vue.config.js文件，用来进行一些简单配置

    https://cli.vuejs.org/zh/guide/webpack.html

    如：修改默认端口和标题

    //vue.config.js
const port = 7070
module.exports = {
    publicPath:	'/best-practice', //配置路径的上下文（url前面会加上这个）
    devServer:{
        port
    },
    configureWebpack:{
        name:title        //然后到index.html中在title标签内部写上<%= webpackConfig.name %>
    }
}

# webpack就是模块打包工具

Webpack可以看做是模块打包机：它做的事情是，分析你的项⽬目结构，找到JavaScript模块以及其它的⼀一些浏览器器不不能直接运⾏行行的拓拓展语⾔言（Scss，TypeScript等），并将其打包为合适的格式以供浏览器器使⽤用。webpack 只能理解 JavaScript 和 JSON 文件。

  • 默认配置文件：webpack.config.js

执行·webpack命令或者npx webpack打包后，webpack会找到默认配置文件，并执行使用

不使用默认的配置文件就用 --config指定使用哪个配置文件

webpack --config myWebpackConfig.js //或使用
npx webpack --config myWebpackConfig.js //来指定作为配置文件来使用

# webpack.config.js 一般配置结构

module.exports = {
    mode:'development'  //打包的环境
    entry:'', 			//打包入口文件
    output:{			//输出
        path:path.resolve(__dirname,'dist'),
        filename:'bundle.js'
    },
    module:{
        rules:[
            //下面是用正则匹配不同的路径，在打包之前，先用对应的loader先处理
            {
                test:/\.css$/,	
                use:['style-loader','css-loader']  //loader顺序，从后往前
            }
        ]
    },
    plugins:[
        new HtmlWebpackPlugin()   //插件配置
    ]
}

# webpack核心概念

entry: 指定webpack打包入口文件，webpack执行构建的第一步从entry开始，理解为输入

//入口：单入口，多入口
entry:{
    main:'./src/index.js'
}
//可简写
entry:'./src/index.js'

//多入口，entry为对象
entry:{
    index:'./src/index.js'
    login:'./src/login.js'
}
